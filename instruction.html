Reverse-C Programming Language


1. Using the Reverse-C Transpiler

The Reverse-C Transpiler is a comical language that takes 0s and 1s in multiples of 8 while disregarding white space and newlines to produce code. The intent is to create a programs that looks like binary and produces C functional C code in a file called CFile.c.

The Reverse-C Transpiler recognizes 17 instructions
Each instruction uses either a variable, a number, or a label as operands. 
  • variable - this corresponds to a memory location that can be used to hold a value
  • number - this corresponds to any unsigned number that can be represented as an eight digit binary number
  • label - this corresponds to a memory location that can be referenced corresponding to some instruction
Each operand used within an instruction is either a source (src) or destination (dest) 
  • dest - this corresponds to a memory location where data is stored
  • src - this corresponds to a memory location where data is retrieved
Below are all of the defined instructions with descriptions 
  • Set value - Place any byte sized number into the dest memory location
instruction	dest - variable    src - number
00000001 	00000001	   00000010

  • Move value - Change the memory location of the reference by a byte amount
instruction	dest - variable    src - number
00000010 	00000001	   00000010

  • Set value using variable - Place the contents of the src location into the dest memory location
instruction	dest - variable    src - variable
00000011 	00000001	   00000010

  • Is variable equal to another variable - Check if the two src variable contents are equal and place the result in the dest variable
instruction	dest - variable    src1 - variable	src2 - variable
00000100 	00000001	   00000010	        00000100

  • Is variable greater than another variable - Check if src1 variable contents are greater than src2 and place the result in the dest variable
instruction	dest - variable    src1 - variable	src2 - variable
00000101 	00000001	   00000010	        00000100

  • Is variable lesser than another variable - Check if src1 variable contents are lesser than src2 and place the result in the dest variable
instruction	dest - variable    src1 - variable	src2 - variable
00000110 	00000001	   00000010	        00000100

  • Is equal then jump - If the two src variable contents are equal then jump to the specified label
instruction	src1 - variable    src2 - variable	label
00000111 	00000001	   00000010	        00000100

  • Is greater then jump - if src1 variable contents are greater than src2 then jump to the specified label
instruction	src1 - variable    src2 - variable	label
00001000 	00000001	   00000010	        00000100

  • Is lesser then jump - if src1 variable contents are lesser than src2 then jump to the specified label
instruction	src1 - variable    src2 - variable	label
00001001 	00000001	   00000010	        00000100

  • Add variables - Add src1 variable contents to src2 contents and place the result in the dest variable
instruction	dest - variable    src1 - variable	src2 - variable
00001010 	00000001	   00000010	        00000100

  • Subtract variables  - Subtract src2 variable contents from src1 contents and place the result in the dest variable
instruction	dest - variable    src1 - variable	src2 - variable
00001011 	00000001	   00000010	        00000100

  • And variables - Perform a logical "and" operation on src1 variable contents and src2 contents and place the result in the dest variable
instruction	dest - variable    src1 - variable	src2 - variable
00001100 	00000001	   00000010	        00000100

  • Or variables - Perform a logical "or" operation on src1 variable contents and src2 contents and place the result in the dest variable
instruction	dest - variable    src1 - variable	src2 - variable
00001101 	00000001	   00000010	        00000100

  • Set label - Set a label that can be used a memory reference for jump instructions
instruction	label
00001110 	00000001

  • Jump to label - Jump to any already set label
instruction	label
00001111 	00000001

  • print number - print the contents of a memory location interpreted as a number
instruction	src - variable
00010000 	00000001

  • print char - print the contents of a memory location interpreted as a character
instruction	src - variable
00010001 	00000001
While Reverse-C does handle the allocation and deallocation of memory for the user, it does not guarantee that all memory interactions are safe or logically meaningful. Ensuring program correctness and safety largely lies with the user as this project is not intended to produce robust programmatic solutions.
